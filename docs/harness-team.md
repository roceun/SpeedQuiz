# 하네스 팀 운영 문서

## 목적

이 팀은 "초등학생도 바로 이해하고 즐길 수 있는 스피드 게임"을 만드는 것을 목표로 움직입니다.

게임의 핵심은 아래와 같습니다.

- 주제를 고른다
- 랜덤 단어 10개가 한 개씩 나온다
- 한 명은 설명하고 한 명은 맞힌다
- 턴이 끝나면 맞은 개수와 틀린 개수를 보여준다
- 다음 턴에는 역할을 바꿔서 진행한다

## 팀 구성

### 1. 기획자

역할

- 게임 규칙을 단순하고 명확하게 정리
- 초등학생에게 익숙한 단어와 주제를 선정
- 한 턴의 흐름, 점수 표시, 역할 교대 방식을 설계

산출물

- 게임 규칙 초안
- 화면 흐름 초안
- 단어 카테고리 초안

체크 기준

- 설명을 듣는 즉시 이해할 수 있는가
- 글이 너무 많거나 어렵지 않은가
- 승패보다 재미와 템포가 잘 살아 있는가

### 2. 개발자

역할

- 기획 문서를 바탕으로 UI와 게임 로직 구현
- 랜덤 단어 선택, 턴 진행, 결과 집계 기능 구현
- 이후 확장 가능한 구조로 프로젝트 기반 정리

산출물

- 실행 가능한 게임 화면
- 주제 데이터 구조
- 턴/점수 로직

체크 기준

- 버튼과 흐름이 단순한가
- 한 턴 종료 시 결과가 정확하게 집계되는가
- 역할 교대가 헷갈리지 않게 보이는가

### 3. 검수자

역할

- 기획과 구현이 초등학생 기준에 맞는지 검토
- 문구, 버튼, 진행 순서의 혼란 요소 점검
- 버그와 사용성 문제를 구분해서 전달

산출물

- 시나리오별 점검 기록
- 버그 목록
- 개선 제안 목록

체크 기준

- 처음 보는 아이도 1분 안에 시작할 수 있는가
- 오답/정답 처리 방식이 직관적인가
- 턴이 끝났을 때 결과를 쉽게 이해할 수 있는가

## 협업 흐름

1. 기획자가 한 기능 단위를 짧게 정의한다.
2. 개발자가 바로 구현 가능한 작업으로 나눈다.
3. 검수자가 아이 기준으로 읽고 눌러보며 문제를 찾는다.
4. 수정 후 다시 같은 흐름으로 확인한다.

## 현재 공통 원칙

- 초등학생이 읽기 쉬운 짧은 문장을 사용한다.
- 복잡한 설정 없이 바로 시작할 수 있게 만든다.
- 경쟁보다 즐거운 상호작용에 집중한다.
- 한 화면에서 현재 상태를 최대한 쉽게 이해할 수 있게 만든다.

## 다음 작업 제안

다음 단계에서는 기획자가 아래 내용을 먼저 확정합니다.

- 주제 목록: 동물, 자연, 사물 등
- 한 턴 진행 방식: 시간제 또는 10문제 순차 진행
- 정답/패스/오답 처리 버튼 구성
- 마지막 결과 화면 구성
