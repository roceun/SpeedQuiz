# 스피드 게임 기획 확정본

## 게임 한 줄 설명

주제를 고르고 10개의 단어를 하나씩 보면서 친구와 설명하고 맞히는 간단한 2인용 스피드 게임

## 목표

- 한국 초등학생이 바로 시작할 수 있는 쉬운 게임 만들기
- 복잡한 설정 없이 `시작 -> 게임 -> 결과 보기`로 끝나는 구조 만들기
- 마지막에 맞춘 개수만 간단하게 확인할 수 있게 만들기

## 핵심 규칙

1. 시작 화면에서 주제를 고른다.
2. 시간 제한을 고른다. `무제한`, `1분`, `3분`, `5분` 중에서 선택할 수 있다.
3. 선택한 주제에서 단어 10개를 랜덤으로 뽑는다.
4. 한 명은 단어를 보고 설명하고, 다른 한 명은 맞힌다.
5. 맞히면 `정답` 버튼을 누르고, 못 맞히면 `다음` 버튼을 누른다.
6. 10개를 모두 진행하거나 시간이 끝나면 결과 화면으로 이동한다.
7. 결과 화면에서는 맞춘 개수만 보여준다.

## 화면 구성

### 1. 시작 화면

보여줄 내용

- 게임 제목
- 주제 선택
- 시간 제한 선택
- 시작 버튼

화면 목적

- 복잡한 설명 없이 바로 게임을 시작하게 한다.

추천 문구

- `주제를 골라요`
- `시작하기`

### 2. 게임 화면

보여줄 내용

- 현재 단어
- 진행 표시: `현재 몇 번째 단어인지`
- 남은 시간 또는 무제한 표시
- 정답 버튼
- 다음 버튼

화면 목적

- 단어를 하나씩 보여주고 빠르게 넘길 수 있게 한다.
- 누르는 버튼 수를 최소화한다.

버튼 규칙

- `정답`: 친구가 맞힌 경우
- `다음`: 못 맞혔거나 건너뛰고 다음 단어로 넘어가는 경우

### 3. 결과 화면

보여줄 내용

- 총 10개 중 맞춘 개수
- 다시 시작 버튼

화면 목적

- 결과를 아주 짧고 분명하게 보여준다.

추천 문구

- `10개 중 6개 맞혔어요`
- `다시 하기`

## 전체 흐름

1. 시작 화면
2. 게임 화면
3. 결과 화면

## UX 원칙

- 설명 문구를 최소화한다.
- 화면마다 해야 할 행동이 1개 또는 2개만 보이게 한다.
- 버튼 글자는 짧고 쉽게 쓴다.
- 결과는 한눈에 이해되게 만든다.

## 데이터 초안

필요한 데이터

- 선택한 주제
- 선택한 시간 제한
- 랜덤 단어 10개
- 현재 단어 순서
- 맞춘 개수
- 남은 시간
- 게임 종료 여부

## 개발 전달용 결정 사항

- 첫 버전은 2명이 한 기기에서 플레이
- 시간 제한은 `무제한 / 1분 / 3분 / 5분`
- 화면은 `시작 -> 게임 -> 결과 보기` 3단계
- 게임 화면 버튼은 `정답 / 다음`
- 결과 화면은 맞춘 개수만 표시
