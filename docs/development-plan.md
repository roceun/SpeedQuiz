# 스피드 게임 개발 계획

## 개발 목표

기획 문서 [docs/game-plan.md](/Users/roceun/Documents/New project/docs/game-plan.md)를 기준으로, 초등학생이 바로 사용할 수 있는 아주 단순한 웹 게임을 만든다.

이번 개발 단계의 목표는 아래 3가지입니다.

- 빠르게 실행 가능한 첫 버전 만들기
- 구조가 단순해서 수정이 쉬운 상태로 만들기
- 이후 단어 추가나 디자인 수정이 편한 형태로 만들기

## 기술 선택

첫 버전 권장 기술

- 프레임워크: React + TypeScript
- 빌드 도구: Vite
- 스타일링: 일반 CSS 또는 CSS Modules

선택 이유

- 화면이 3개뿐이라 구조가 단순하다.
- 상태 관리가 복잡하지 않아 외부 상태 라이브러리가 필요 없다.
- TypeScript를 쓰면 단어 데이터와 게임 상태를 안전하게 다룰 수 있다.
- Vite는 초기 세팅과 실행 속도가 빠르다.

이번 버전에서 사용하지 않는 것

- Redux, Zustand 같은 전역 상태 라이브러리
- 서버 API
- 데이터베이스
- 로그인 기능
- 애니메이션 라이브러리

## 구현 대상

반드시 구현할 기능

- 시작 화면
- 주제 선택 기능
- 선택한 주제에서 단어 10개 랜덤 추출
- 게임 화면에서 단어 1개씩 표시
- `정답` 버튼 클릭 시 맞춘 개수 증가
- `다음` 버튼 클릭 시 다음 단어로 이동
- 10개 진행 후 결과 화면 표시
- 결과 화면에서 맞춘 개수 표시
- 다시 시작 기능

이번 단계에서 구현하지 않는 기능

- 플레이어 이름 입력
- 턴 교대
- 시간 제한
- 점수 비교
- 사운드
- 저장 기능

## 화면별 개발 정의

### 1. 시작 화면

필요 요소

- 게임 제목
- 주제 선택 UI
- 시작 버튼

동작 조건

- 주제를 선택해야 시작 버튼이 활성화된다.
- 시작 버튼을 누르면 게임이 시작된다.

개발 포인트

- 가장 먼저 보이는 화면이어야 한다.
- 선택 상태가 눈에 잘 보여야 한다.

### 2. 게임 화면

필요 요소

- 현재 주제 이름
- 현재 단어
- 진행 상태: `현재 순서 / 10`
- 맞춘 개수
- `정답` 버튼
- `다음` 버튼

동작 조건

- `정답` 클릭 시 맞춘 개수를 1 올리고 다음 단어로 이동
- `다음` 클릭 시 점수 변화 없이 다음 단어로 이동
- 10번째 단어 처리 후 결과 화면으로 이동

개발 포인트

- 현재 단어가 가장 크게 보여야 한다.
- 버튼은 손가락으로 누르기 쉽게 크게 만든다.
- 현재까지 몇 개 맞췄는지 계속 보이게 한다.

### 3. 결과 화면

필요 요소

- 맞춘 개수
- 다시 하기 버튼

동작 조건

- 다시 하기 버튼을 누르면 초기 상태로 돌아간다.

개발 포인트

- 결과는 한 문장으로 바로 이해되게 보여준다.
- 복잡한 통계는 넣지 않는다.

## 상태 설계

첫 버전 상태는 컴포넌트 최상단에서 한 번에 관리합니다.

필요 상태

- `screen`: `start | play | result`
- `selectedCategory`: 현재 선택한 주제
- `words`: 이번 게임에 사용될 랜덤 단어 10개
- `currentIndex`: 현재 보고 있는 단어 위치
- `correctCount`: 맞춘 개수

상태 전이

1. 시작 화면: 주제를 선택한다.
2. 시작 버튼 클릭: 단어 10개를 준비하고 `play`로 이동한다.
3. 게임 진행: 버튼 클릭마다 `currentIndex`가 증가한다.
4. 마지막 단어 이후: `result`로 이동한다.
5. 다시 하기 클릭: 모든 상태를 초기값으로 되돌린다.

## 데이터 구조

주제 데이터는 정적 파일로 분리합니다.

예상 구조

```ts
type CategoryKey = "animals" | "nature" | "objects" | "foods" | "places";

type Category = {
  key: CategoryKey;
  label: string;
  words: string[];
};
```

예상 데이터 예시

```ts
export const categories: Category[] = [
  {
    key: "animals",
    label: "동물",
    words: ["강아지", "고양이", "토끼"],
  },
];
```

## 추천 파일 구조

```text
src/
  App.tsx
  main.tsx
  styles.css
  data/
    categories.ts
  components/
    StartScreen.tsx
    PlayScreen.tsx
    ResultScreen.tsx
  utils/
    shuffle.ts
```

파일 역할

- `App.tsx`: 전체 상태 관리와 화면 전환
- `data/categories.ts`: 카테고리 및 단어 데이터
- `StartScreen.tsx`: 주제 선택과 시작 버튼
- `PlayScreen.tsx`: 단어 진행 화면
- `ResultScreen.tsx`: 결과 화면
- `utils/shuffle.ts`: 단어 랜덤 섞기

## 핵심 로직 정의

### 1. 랜덤 단어 추출

- 선택한 카테고리의 전체 단어를 섞는다.
- 앞에서 10개를 잘라서 사용한다.
- 한 게임 안에서는 중복 없이 보여준다.

### 2. 정답 처리

- `정답` 클릭
- `correctCount + 1`
- 다음 단어로 이동

### 3. 다음 처리

- `다음` 클릭
- 점수 변화 없음
- 다음 단어로 이동

### 4. 게임 종료 처리

- 현재 단어가 마지막 단어라면 결과 화면으로 이동
- 아니라면 다음 단어 표시

## UI 방향

첫 버전 UI 원칙

- 밝고 친근한 색감 사용
- 버튼은 크게, 텍스트는 짧고 굵게
- 현재 단어를 화면 중심에 크게 표시
- 주제 선택은 카드나 큰 버튼 형태로 표시

기본 화면 우선순위

1. 지금 해야 할 행동
2. 현재 단어 또는 결과
3. 보조 정보

## 테스트 체크리스트

- 주제를 선택하지 않으면 시작되지 않는가
- 게임 시작 시 단어가 10개 준비되는가
- `정답`을 누르면 점수가 올라가는가
- `다음`을 누르면 점수는 그대로인가
- 10개 뒤에 결과 화면으로 넘어가는가
- 결과 화면에 맞춘 개수만 보이는가
- 다시 하기를 누르면 처음으로 돌아가는가
